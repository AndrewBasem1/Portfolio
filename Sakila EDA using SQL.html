Exploratory Data Analysis of the opensource Sakila Database Using SQL#

The Sakila Database is a free synthetic database modelling an international DVD renting chain store

We will try to find the answers to the following 3 questions:

  1. Currently rented DVDs:

    1. How many rentals do we have any currently rented by active or inactive users?

    2. If yes, Are there any active customers who might not return those DVDs? (First time customers without any previous successful returns) 3.Provide a list of customers in possesion of rentals that have a high probaiblity of not returning them

#a simple function to read an SQL query from the database into a dataframe, parse dates when needed, and returns the df
def DisplayData(query):
    cols = pd.read_sql(query,con).columns
    date_cols = []
    for col in cols:
        if 'date' in col or 'last_update' in col:
            date_cols.append(col)
    df = pd.read_sql(query,con,parse_dates=date_cols)
    return(df)

How many rentals do we have any currently rented by active or inactive users?

Querying the “rental” and “customer” table to get a representation of the distribution of outstanding DVDs across active and inactive Users

SELECT CASE
            WHEN c.active = 0 THEN "Inactive Customer(s)"
            WHEN c.active = 1 THEN "Active Customer(s)"
            END AS is_active
        , Count(r.customer_id) AS total_outstanding_rentals
    From rental as r, customer as c
    WHERE return_date is NULL
          AND c.customer_id = r.customer_id
    group by is_active;

If yes, Are there any active customers who might not return those DVDs? (First time customers without any previous successful returns)

Building two CTEs based on the “rental” table, when joined together with the “customers” table we can find out the number of previous successful rentals for each active customer

WITH outstanding AS (
    SELECT customer_id,count(customer_id) as outstanding_rentals
            From rental
            WHERE return_date is NULL
            GROUP BY customer_id
)
, returned AS (
    SELECT customer_id,count(customer_id) as succussful_returns
            From rental
            WHERE return_date is not NULL
            GROUP BY customer_id
)

SELECT Distinct r.customer_id, o.outstanding_rentals, r.succussful_returns
    FROM returned as r
    INNER JOIN customer AS c
        ON c.customer_id = r.customer_id AND c.active = 1
    INNER JOIN outstanding AS o
        ON c.customer_id = o.customer_id
    ORDER BY succussful_returns ASC;

It’s Evident that all active customers have at least 18 previous successful rents that they have returned, so it’s not probable that they will not return the currently rented items.

# Checking the number of prvious successful returns for customers with current outstanding rentals
query = """
WITH outstanding AS (
    SELECT customer_id,count(customer_id) as outstanding_rentals
            From rental
            WHERE return_date is NULL
            GROUP BY customer_id
), customer_data AS (
    SELECT cu.customer_id,
           cu.first_name,
           cu.last_name,
           cu.email,
           a.address,
           ci.city,
           co.country
    FROM customer AS cu
    LEFT JOIN address AS a
        ON cu.address_id = a.address_id
    LEFT JOIN city AS ci
        ON a.city_id = ci.city_id
    LEFT JOIN country AS co
        ON co.country_id = ci.country_id
    WHERE cu.active = 0
)

SELECT cd.*, o.outstanding_rentals
FROM customer_data AS cd, outstanding as o
where o.customer_id = cd.customer_id
ORDER BY o.outstanding_rentals DESC
"""
DisplayData(query)
